## Color Scheme

- **Primary Color**: iOS System Blue (#007AFF)
- **Success Color**: Green
- **Error Color**: Red
- **Background**: White (Light Mode) / Black (Dark Mode)
- **Input Fields**: Light Gray Background (#F2F2F7)
- **Text**: System Default (Black/White based on mode)

## Interactive States

### Buttons
- **Normal**: Blue background, white text
- **Disabled**: Gray background, white text
- **Loading**: Shows circular progress indicator

### Input Fields
- **Empty**: Light gray background, placeholder text
- **Focused**: Blue border appears
- **Error**: Red text below field

## Animations

- Smooth transitions between views (NavigationStack)
- Fade in/out for error/success messages
- Progress indicators for async operations
- Alert modal with slide-in animation

## Accessibility

All views support:
- Dynamic Type (text scaling)
- VoiceOver labels
- Keyboard navigation
- Dark Mode

## Notes

To see the actual UI:
1. Open the project in Xcode
2. Select a simulator (iPhone 15 Pro recommended)
3. Build and run (Cmd+R)
4. The app will show the AuthenticationView as the landing screen

### Sorting Options

The segmented control provides three sorting options:

1. **Recent** (Default) - Shows newest posts first (sorted by creation time, descending)
2. **Most Likes** - Shows posts with the most likes first (sorted by like count, descending)
3. **Oldest** - Shows oldest posts first (sorted by creation time, ascending)

**User Interaction:**
- Tap any segment to change sort order
- Posts automatically reload with the selected sort order
- Pull down to refresh maintains the current sort selection
- Sort preference is maintained while on the same view

**Visual States:**
- Selected segment: Blue background with white text
- Unselected segments: White background with black text
- Smooth animation when switching between segments

### Profile Sorting Options

**Posts Sorting:**
1. **Recent** (Default) - Shows newest posts first (sorted by creation time, descending)
2. **Most Likes** - Shows posts with the most likes first (sorted by like count, descending)
3. **Oldest** - Shows oldest posts first (sorted by creation time, ascending)

**Comments Sorting:**
1. **Recent** (Default) - Shows newest comments first (sorted by creation time, descending)
2. **Oldest** - Shows oldest comments first (sorted by creation time, ascending)

Note: Comments do not have a "Most Likes" option since comments don't have like functionality.

**User Interaction:**
- Switch between Posts and Comments using the top segment control
- Each tab displays a "Sort by:" dropdown menu
- Tap the dropdown button to see available sorting options
- The currently selected option shows a checkmark (âœ“)
- Tap a different option to change the sort order
- Content automatically reloads with the selected sort order
- The dropdown button displays the currently selected sort option
- Each tab maintains its own sort preference independently
- Pull down to refresh maintains the current sort selection

**Visual Design:**
- Dropdown button with gray background (Color(.systemGray6))
- Blue text and chevron-down icon
- Menu options with checkmark for current selection
- Clean, compact design that doesn't interfere with the Posts/Comments segmented control
- Avoids confusion by using a different UI pattern than the segment control

The UI is built entirely with SwiftUI and follows iOS Human Interface Guidelines for a native iOS experience.
